# ai-toir — семантический поиск похожих дефектов

## Как это работает

1. **Офлайн-индексация** — `build_cache.py` читает CSV с дефектами и
   считает эмбеддинги поля «Описание дефекта» моделью **BAAI/bge-m3**
   (1024-dim, 15 436 записей, ≈335 МБ JSON, L2-нормализованные).
2. **HTTP-сервер** — `serve.py` отдаёт статику (`index.html`,
   `defects_cache.json`) и эндпоинт `POST /embed` для векторизации
   пользовательского запроса той же моделью.
3. **Браузер** — `index.html` загружает кэш, считает косинусное сходство
   с запросом в JavaScript (≈100 мс на 15 436 записей), показывает топ-K
   с фильтрами по АЭС и Оборудованию.

## Запуск

### 1. Подготовьте CSV

Положите файл `Описания_дефектов.csv` (кодировка CP1251, разделитель `;`)
в корень проекта. **Этот файл не публикуется в репо** (конфиденциальные
данные). Ожидаемые колонки: `Номер`, `Описание дефекта`, `Причина`,
`Оборудование`, `Завод`, `Дата`.

### 2. Соберите кэш эмбеддингов

```bash
pip install FlagEmbedding
python download_bge_m3.py    # один раз: ≈2.2 ГБ в ~/.cache/huggingface
python build_cache.py        # ≈33 мин на CPU
```

Получите `defects_cache.json` (≈335 МБ).

### 3. Запустите сервер и откройте UI

```bash
python serve.py
# → http://localhost:8000/index.html
```

Для доступа из локальной сети — добавьте правило Windows Firewall
(порт 8000, Inbound, Allow), и коллеги откроют `http://<ваш-IP>:8000/index.html`.

## Файлы

| Файл | Что делает |
|---|---|
| `build_cache.py` | CSV → эмбеддинги BAAI/bge-m3 → `defects_cache.json` |
| `download_bge_m3.py` | Докачка модели BAAI/bge-m3 в кэш HF (≈2.2 ГБ) |
| `serve.py` | HTTP-сервер: статика + `POST /embed` (port 8000) |
| `index.html` | UI: тёмная тема, фильтры АЭС/Оборудование, таблица |
| `defects_cache.json` | (не в репо) 15 436 записей × 1024-dim, ≈335 МБ |
| `Описания_дефектов.csv` | (не в репо) исходные данные |

## Особенности

### FlagEmbedding + `show_progress_bar`

В `build_cache.py` **нельзя** передавать `show_progress_bar=True` в
`FlagModel.encode(...)` — `FlagEmbedding` прокидывает `**kwargs` напрямую
в `self.tokenizer.pad(...)`, а современные `transformers`/`tokenizers`
не принимают этот аргумент. Без него всё работает.

### Перезапуск `serve.py`

Перед перезапуском **обязательно убить старый процесс** — иначе `/embed`
отвечает от старого процесса со старой (другой) моделью, и сходства
не сходятся. Проверка:

```powershell
Get-Process python
# или через curl: должен вернуть 1024-dim массив
(Invoke-WebRequest -Uri "http://localhost:8000/embed" -Method POST `
  -ContentType "application/json" -Body '{"text":"тест"}' -UseBasicParsing).Content
```

### Длинные значения в фильтрах

`index.html` обрезает длинные названия в `<select>`-ах (АЭС, Оборудование)
до 16 символов с `…` — иначе native `<select>` в Chrome «вылезает» за
границы колонки. Полное значение — в `title` (всплывающая подсказка) и
в выпадающем списке.

### Зависимые фильтры

Списки АЭС и Оборудование в UI обновляются **по результатам поиска**
(не по всему кэшу): после клика «Найти» в обоих фильтрах остаются
только те значения, которые встретились среди найденных дефектов. Если
выбранное значение пропало — фильтр сбрасывается на «Все» и поиск
перезапускается автоматически.

## Требования

- Python 3.10+
- `FlagEmbedding` (тянет `torch`, `transformers`, `huggingface_hub`)
- 4+ ГБ RAM (для BAAI/bge-m3 в fp32)
- 3 ГБ на диске (модель + кэш эмбеддингов)
- Современный браузер с поддержкой `fetch` и `IndexedDB`
